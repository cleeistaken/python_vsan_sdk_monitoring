Storage Policy Data Object Overview

The Storage Policy data objects contain information about storage policies, profiles, and capabilities. You use managed object methods to retrieve data objects and you specify data objects when you call methods.

The ManagedObjectReference data object type is a special-purpose data object. Commonly referred to as simply a “reference, ” “MoRef,” “MOR,” or other variations of this theme, instances of managed object references contain data that identifies specific server-side managed objects. Most method invocations require a managed object reference as a parameter. A reference can be embedded in a data object and methods can return references.

All data objects are listed in the frame at the left of this page. Click a name to display the reference documentation for the data object. Reference documentation for data objects typically includes:

  • The data object type from which it extends.
  • Descendent data objects, if any, that extend the base data object type.
  • Description of purpose and usage.
  • Names of properties and their data types that comprise the data object (the members of the data object type).

To find a data object, start typing its name in the Quick Index.
